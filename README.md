# מונופול כרטיס אשראי – בנק אלקטרוני

יחידת הבנקאות האלקטרונית למשחק מונופול כרטיס אשראי (Monopoly Electronic Banking).  
ממשק בעברית (RTL), מותאם למובייל.

## טכנולוגיות

- **Next.js 14** (App Router), **React**, **Tailwind CSS**, **Supabase** (מסד נתונים + Real-time)

## התקנה

1. **Clone והתקנת תלויות**
   ```bash
   cd "Monopoly Game"
   npm install
   ```

2. **Supabase**
   - צור פרויקט ב־[Supabase](https://supabase.com).
   - ב־SQL Editor הרץ את הקובץ `supabase/schema.sql` (כולו) כדי ליצור טבלאות, RLS ו־Real-time.

3. **משתני סביבה**
   - העתק `.env.local.example` ל־`.env.local`.
   - הזן:
     - `NEXT_PUBLIC_SUPABASE_URL` – כתובת הפרויקט.
     - `NEXT_PUBLIC_SUPABASE_ANON_KEY` – מפתח anon (public).

4. **הרצה**
   ```bash
   npm run dev
   ```
   גלוש ל־`http://localhost:3000`.

## כללים (תואם למשחק הרשמי)

- **מטבע:** ש"ח. תצוגה: M למיליונים (למשל 1.5M), K לאלפים (למשל 500K).
- **יתרה התחלתית:** 15M (15,000,000) לכל שחקן.
- **דרך צלחה:** פעולה מהירה שמוסיפה 2M מהבנק לשחקן.
- **גבולות:** מינימום העברה 0, מקסימום פעולה בודדת 20M.

## זרימת האפליקציה

1. **דף נחיתה:** צור חדר (שם) או הצטרף (קוד חדר 4–6 ספרות + שם).
2. **מסך משחק:** קוד חדר, שם שחקן, יתרה; כפתורי פעולה; היסטוריית פעולות מעודכנת בזמן אמת.

## מבנה הפרויקט

- `src/app/` – דפים (נחיתה, חדר משחק).
- `src/app/room/[code]/` – דשבורד משחק, פעולות ומודלים, ledger.
- `src/lib/` – Supabase client, טיפוסים, פונקציות מטבע.
- `supabase/schema.sql` – סקריפט SQL להרצה ב־Supabase.

אין אימות – גישה לפי קוד חדר ושם בלבד.
